XPath Practices for Reliable Automation

Short, practical tips to build stable selectors that survive UI changes.

1. Prefer IDs and stable attributes

Use unique IDs when available. If not, prefer stable attributes like data-testid or aria-label.

2. Avoid deep absolute paths

Absolute paths like /html/body/div[2]/... break easily. Use shorter, attribute-based selectors instead.

3. Use text selectors carefully

Text-based XPaths are useful for labels and buttons. Keep them short and normalized to avoid spacing issues.

4. Verify uniqueness

Always validate that your XPath matches exactly one element to avoid flaky tests.
